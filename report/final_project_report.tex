\section{REINFORCE with Baseline algorithm}
\subsection{Brief Description}
REINFORCE is a Monte Carlo policy gradient algorithm that optimizes parameterized policies to maximize expected total return. 
The agent interacts with the environment in episodes, collecting trajectories of states, actions, and rewards. 
After each episode, the agent updates its policy parameters based on the observed returns using gradient ascent.
$$\bm{\theta}_{t+1} = \bm{\theta}_t + \alpha G_t \frac{\nabla \pi(A_t|S_t, \bm{\theta}_t)}{\pi(A_t|S_t, \bm{\theta}_t)}$$
where $G_t$ is the return following time step $t$, $\alpha$ is the learning rate, and $\pi(a|s,\bm{\theta})$ is the policy parameterized by $\bm{\theta}$.
\\
\\
The REINFORCE with Baseline algorithm introduces a baseline function to the REINFORCE algorithm so that the equation becomes:
$$\bm{\theta}_{t+1} = \bm{\theta}_t + \alpha (G_t - b(S_t)) \frac{\nabla \pi(A_t|S_t, \bm{\theta}_t)}{\pi(A_t|S_t, \bm{\theta}_t)}$$
where $b(S_t)$ is the baseline function that depends only on the state $S_t$.
The baseline function can be an function as long as it does not depend on the action and varies with states for MDPs. 
One natural choice is for baseline to be an estimate of the state-value function $\hat{v}(S_t, \bm{w})$ where $\bm{w}$ is the state-value weights that can be learned using Monte Carlo methods:
$$\bm{w}_{t+1} = \bm{w}_t + \alpha^{\bm{w}} (G_t - \hat{v}(S_t, \bm{w}_t)) \nabla \hat{v}(S_t, \bm{w}_t)$$

\section{One-Step Actor-Critic algorithm}
\subsection{Brief Description}

\paragraph{One-step Actor-Critic Algorithm.}
The One-step Actor-Critic algorithm is an on-policy reinforcement learning method that jointly learns a policy and a state-value function. The policy $\pi(a | s, \bm{\theta})$, called the actor, is a differentiable function parameterized by $\bm{\theta}$ and defines a probability distribution over actions given a state. The state-value function $\hat v(s,\bm{w})$, called the critic, is also differentiable and is parameterized by $\bm{w}$, estimating the expected discounted return from a state under the current policy. The algorithm uses separate step sizes $\alpha^{\bm{\theta}}$ and $\alpha^{\bm{w}}$ for updating the actor and the critic, respectively.

\paragraph{Episode Initialization.}
At the beginning of each episode, the environment is reset to obtain the initial state $S$. The auxiliary variable $I$ is initialized to $1$ and is used to apply discounting in the policy gradient update. The parameters $\bm{\theta}$ and $\bm{w}$ are initialized before training, typically to zero or small random values.

\paragraph{Interaction and TD Error.}
At each time step, while the current state $S$ is not terminal, an action $A$ is sampled from the current policy $\pi(\cdot | S, \bm{\theta})$. After executing the action, the agent observes a reward $R$ and the next state $S'$. The temporal-difference (TD) error is then computed as
\[
\delta = R + \gamma \hat v(S', \bm{w}) - \hat v(S, \bm{w})
\]
where $\gamma$ is the discount factor. If the next state $S'$ is terminal, the value estimate $\hat v(S', \bm{w})$ is set to zero. The TD error measures the difference between the observed return and the current value estimate.

\paragraph{Critic Update.}
The critic updates the value-function parameters using one-step TD learning:
\[
\bm{w} \leftarrow \bm{w} + \alpha^{\bm{w}} \, \delta \, \nabla \hat v(S, \bm{w})
\]
This update reduces the prediction error of the value function by moving $\hat v(S,\bm{w})$ toward the one-step bootstrapped target.

\paragraph{Actor Update.}
The actor updates the policy parameters using a policy gradient step:
\[
\bm{\theta} \leftarrow \bm{\theta} + \alpha^{\bm{\theta}} \, I \, \delta \, \nabla \ln \pi(A | S, \bm{\theta})
\]
We also have
\[\nabla \ln \pi(a | s, \bm{\theta}) = \frac{\nabla \pi(a | s, \bm{\theta})}{\pi(a | s, \bm{\theta})} = \text{\textbf{x}}(s,a) - \sum_{b} \pi(b | s, \bm{\theta}) \text{\textbf{x}}(s,b)\]
where $\text{\textbf{x}}(s,a)$ is the feature vector for state-action pair $(s,a)$. 

\subparagraph{Polynomial Feature.}
Polynomial features represent the state by using polynomial terms of the original state variables. Given a state vector $s = (s_1, s_2, \dots, s_d)$, the feature vector may include constant terms, single variables, squared or higher-order terms, and cross-products such as $s_i s_j$. This allows a linear function approximator to model nonlinear relations between the state and the value or policy. However, the number of features grows quickly as the polynomial degree or the state dimension increases, which can increase computation cost and risk overfitting.

\subparagraph{Fourier Feature.}
Fourier features map a continuous state into sinusoidal basis functions. For a normalized state $s \in [0,1]^d$, a typical feature has the form $\cos(\pi k \cdot s)$ or $\sin(\pi k \cdot s)$, where $k$ is an integer vector that controls the frequency. These features are global, meaning that a change in the state affects many feature values at the same time. Fourier features are well suited for approximating smooth value functions and are often used in reinforcement learning with continuous state spaces.

\subparagraph{Tile Coding Feature.}
Tile coding is a sparse and binary feature representation. The state space is divided into multiple overlapping grids called tilings, each with a different offset. For a given state, exactly one tile in each tiling is active, and its corresponding feature value is set to one, while all other features are zero. As a result, similar states share some active tiles, leading to local generalization. Tile coding is simple, efficient, and commonly used with linear value function approximation.

\subparagraph{RBF Feature.}
Radial Basis Function (RBF) features are defined using distance-based functions centered at selected points in the state space. Each feature typically has a Gaussian form,
\[
\phi_i(s) = \exp\left(-\frac{\|s - c_i\|^2}{2\sigma_i^2}\right),
\]
where $c_i$ is the center and $\sigma_i$ controls the width. The feature value is high when the state is close to the center and small when it is far away. RBF features provide smooth and local generalization, but their performance depends on the choice of centers and widths.

\subparagraph{Coarse Coding Feature.}
Coarse coding represents the state using a set of overlapping regions in the state space. Each feature corresponds to one region and is active if the state lies inside that region. The regions are large, so each feature gives low-resolution information about the state. Multiple overlapping active features together provide generalization across nearby states. Coarse coding is conceptually similar to tile coding but is less structured and often described at a more general level.

\paragraph{Transition and Termination.}
After the updates, the discounting variable is updated as $I \leftarrow \gamma I$, and the state is updated as $S \leftarrow S'$. This process repeats until a terminal state is reached, at which point the episode ends. Through repeated episodes, the critic improves its value estimates, and the actor learns a policy that approximates the optimal policy.

    \textbf{Input:} Differentiable state-value function $\hat v(s,\bm{w})$ \\

\subsection{Hyperparameter Tuning}

We systematically tuned hyperparameters for four environments (CartPole, Blackjack, FrozenLake, and Taxi). For each environment, we tested different combinations of learning rates for the actor ($\alpha_\theta$) and critic ($\alpha_w$), feature representations (Polynomial, Fourier, Tile Coding, RBF, and Coarse Coding), and feature complexity (such as polynomial order, number of tilings, or number of centers).  
Each environment was evaluated under four configurations. We analyzed final performance and the stability of the temporal-difference (TD) error.

\subsubsection{Cart Pole}

\paragraph{Configurations Tested}
\begin{center}
\begin{tabular}{c c c c c c}
\hline
Config & Feature & $\alpha_\theta$ & $\alpha_w$ & Params & Final Reward \\
\hline
1 & Polynomial & 0.001 & 0.01 & order=2 & 29.11 \\
2 & Polynomial & 0.002 & 0.02 & order=3 & 29.55 \\
3 & Fourier & 0.001 & 0.01 & order=2 & \textbf{500.00} \\
4 & RBF & 0.001 & 0.01 & centers=4, $\sigma=0.3$ & \textbf{500.00} \\
\hline
\end{tabular}
\end{center}

\paragraph{Key Findings}
Fourier and RBF features both achieved perfect performance, while polynomial features failed to converge. Fourier features converged faster, reaching the maximum reward after 1 episode, whereas RBF required about 700 episodes. Polynomial features oscillated heavily and never exceeded a reward of 100.

Here we find that for Fourier features, the reward curve quickly rises to the maximum of 500. Given the low dimensionality and high determinism of CartPole, as well as the strong approximation capability of the Fourier basis (order 2), this behavior is considered expected — the algorithm rapidly converges to a stable policy.

\paragraph{Analysis}
Low-order polynomial features were not expressive enough to model the nonlinear balance dynamics of CartPole. TD errors remained large and unstable, indicating a non-convergent critic. In contrast, Fourier features captured the oscillatory nature of the pole angle effectively, leading to stable learning and near-zero TD errors after convergence. RBF features also achieved stable performance but required more exploration due to localized generalization.

The main takeaway is that, for CartPole, feature representation dominated learning rate selection. Poor features could not be compensated by tuning $\alpha_\theta$ or $\alpha_w$.

\subsubsection{General Insights}

\paragraph{Feature Representation}
Across all environments, feature choice had a much larger impact than learning rate tuning. Mismatched representations led to large performance gaps that could not be recovered through hyperparameter adjustment.

\paragraph{Learning Rates}
Optimal learning rates depended on reward structure. Sparse-reward environments benefited from higher learning rates, while smooth control tasks required smaller values for stable updates.

\paragraph{TD Error Diagnostics}
TD error behavior was a reliable indicator of learning health. Decreasing variance and oscillation around zero signaled convergence, while persistent bias or large variance indicated feature mismatch or instability.

Tile coding and coarse coding both achieved near-optimal performance in FrozenLake. The best configuration, tile coding with $6 \times 4$ tilings, reached a success rate of 1.00 and converged within approximately 1000 episodes. A coarser tile coding scheme ($4 \times 4$) converged more slowly and attained a final reward of 0.98.

\begin{figure}[H]
    \centering
    \includegraphics[width=0.7\textwidth]{report_fig/frozenlake_tuning.png}
    \caption{The plot of Frozen Lake environment with different hyperparameter tuning.}
\end{figure}

Polynomial features performed poorly, with final performance around 0.18 and no clear convergence trend. Correspondingly, TD errors for tile and coarse coding steadily decreased and converged to small values (around $\pm 0.005$), while polynomial features showed biased and unstable TD errors throughout training.

Tile coding converged the fastest, typically reaching its plateau within 500 episodes. Polynomial and RBF features converged slightly later but displayed smooth and stable learning curves.

The failed configuration exhibited high-variance TD errors that did not stabilize over training.
